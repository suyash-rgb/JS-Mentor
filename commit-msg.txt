refactor: cleanup dead code and refine auth flow

This commit cleans up the frontend by removing obsolete components and 
improving the consistency of the trainer authentication flow.

- **Dead Code Cleanup**: Deleted legacy pages (`About.js`, `Testimonials.js`, 
  `Login.js`, `Signup.js`, `search.js`) that were merged into the landing 
  page or replaced by the Clerk/Institute auth flow.
- **Auth Flow Enhancements**: 
  - Updated `ProtectedRoute.js` to allow Trainers to bypass Clerk checks using 
    their custom JWT token, enabling access to the AI and Compiler pages.
  - Standardized "Sign Up Here" terminology across registration entry points.
- **UI Consistency**: Synchronized contact emails in `Home.js` and the Footer 
  to `support@jsmentor.com`.
- **Infrastructure**: Maintained `compiler.js` for legacy support while 
  migrating logic to `jscompiler.js`.
