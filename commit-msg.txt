feat: websocket-based chat system

implemented the WebSocket-based chat system for real-time student-trainer communication.

Key Implementations:
Database Integration (app/models/interaction.py):

Mapped the DoubtReply model to the doubt_replies table.
Added support for image_urls by storing them as a JSON-serialized string in a Text column.
Established relationships so that each chat message is linked to a specific doubt and user (student or trainer).
Authentication Utility (app/dependencies.py):

Created get_user_from_token, a unified validator that handles both Clerk Student JWTs (RS256) and Local Trainer JWTs (HS256).
This allows the WebSocket router to authenticate users via query parameters since custom headers are not supported during the initial WebSocket handshake.
Real-Time Chat Router (app/routers/chat.py):

ConnectionManager: Manages active WebSocket connections grouped by session_id.
Handshake & Auth: Validates the token from query params and ensures the user is either the student or the trainer assigned to the mentorship_session.
Chat History: On connection, it automatically retrieves and sends the last 20 messages from the database so users have immediate context.
Persistence: Every message received is validated (non-empty), saved to the doubt_replies table, and then broadcasted to all active participants in the session.
Robustness: Implemented error handling for broken pipes and abrupt disconnects to prevent memory leaks or server crashes.
Application Entry (app/main.py):

Registered the new chat router with the FastAPI application.
