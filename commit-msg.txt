fix: trainer manual availability control

Availability Endpoint: Added a new PUT endpoint /trainer/me/availability that allows trainers to toggle their is_available status in the database.
Reactive Scheduling Trigger: When a trainer toggles themselves to Online, the backend now automatically triggers the scheduling engine (run_scheduling_engine). This ensures that any pending doubts in the queue are instantly assigned to the newly available trainer without waiting for a student to register a new doubt.
Service Integration: Updated the get_dashboard_overview service to return the current is_available flag so the frontend can initialize the toggle state correctly.
