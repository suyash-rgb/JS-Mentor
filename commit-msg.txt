refactor: consolidate Node.js Learning path into dynamic component with sequential rendering

Summary:
Refactored the Node.js learning path to utilize a single, unified dynamic 
component. This replaces multiple static files and leverages advanced 
iteration logic to ensure content synchronization regardless of JSON 
structure inconsistencies.

Changes:
- Rendering Logic: Enhanced the `renderDynamicSections` helper with a 
  "Sequential Code Counter" to maintain alignment between text headers 
  and code snippets when list-based headers (e.g., Node Modules) are present.
- Regex Optimization: Implemented a "Parent-Child" key filter to prevent 
  description sub-keys (like `title41`) from being incorrectly rendered 
  as primary headings.
- Bug Fixes: 
    - Resolved a 'Double Types' issue where subheadings were duplicating 
      due to greedy regex matching.
    - Fixed an ESLint 'no-undef' error by synchronizing state trackers 
      within the mapping loop.
- UI Consistency: Applied theme-protection CSS and established 
  intermediate-level metadata for the entire Node.js path.

Impact:
- Improved codebase maintainability by reducing file count and 
  standardizing data-mapping patterns.
- Created a robust foundation for real-time exercise injection and 
  upcoming student behavioral analytics.
