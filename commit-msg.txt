fix: Resolve 401 Unauthorized errors for student analytics ingestion

This commit addresses the authentication failure that was blocking the frontend from successfully pushing analytics data to the backend.

1. Clerk JWT Verification
- Installed PyJWT[crypto] to handle RS256 token verification using public keys.
- Implemented `get_current_clerk_student` in `app/dependencies.py` to dynamically fetch Clerk's public JWKS and decode the student's JWT token, rather than relying on the local secret key.
- Updated all POST endpoints in `app/routers/analytics.py` to use the new Clerk-compatible dependency.

2. Temporary Local Dev Workaround (Lazy Upsert)
- Identified that Clerk webhooks cannot hit the localhost backend to create student records upon signup, causing a 403/401 error when the backend attempts to look up the clerk_id.
- Added a TEMPORARY lazy upsert block within `get_current_clerk_student`. If a valid Clerk token is decoded but the user doesn't exist in the local database, it will automatically create a placeholder User and Student profile.
- NOTE: A prominent TODO has been added. This lazy upsert MUST BE REMOVED once the application is deployed to a live environment with functional webhooks.
