fix: Foreign Key Integrity error

In the doubts table, the student_id column is a foreign key that points to the students table's ID. However, the backend code was incorrectly passing the users table's ID (the Clerk user record) into that field. Since those IDs don't match, the database rejected the insertion, resulting in the IntegrityError: (1452, 'Cannot add or update a child row: a foreign key constraint fails') in the logs.

Fixes Applied:
1. Doubt Registration (app/routers/scheduling.py):
 - Updated the register_doubt logic to correctly extract the id from the student's profile (student.student_profile.id) rather than the top-level user object.
     - Added a safety check to ensure a student profile exists before attempting registration, preventing a crash if a non-student user tries to register a doubt.

2. Doubt Retrieval (app/routers/scheduling.py):
 - Applied the same fix to the /doubts/mine endpoint to ensure students only see doubts correctly mapped to their profile ID.
