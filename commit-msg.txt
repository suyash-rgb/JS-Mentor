feat: implement curriculum & exercise management

Summary:
Transitioned curriculum management to a secure, API-driven model. Replaced 
top-level exercise storage with a nested architecture where exercises 
live within specific lesson nodes, enabling contextual hands-on learning.

Changes:
- Security: Implemented `require_trainer` dependency to restrict curriculum 
  modification to authorized users only.
- Discovery: Added `Learning Path Discovery` endpoint to visualize 
  exercise distribution across paths/pages without exposing full content.
- Injection Logic:
    - Added targeted single-exercise injection based on path heading 
      and page text.
    - Implemented `batch-add` capability for efficient multi-exercise 
      uploading in a single IO operation.
- Search & Modify:
    - Developed deep-search traversal for `PUT` and `DELETE` operations, 
      enabling modifications to exercises nested deep within the 
      cards/links hierarchy.
    - Fixed memory-reference bugs by ensuring save_data(data) commits 
      the modified root object to disk.
- Data Integrity: Added duplicate ID validation at the page level to 
  prevent JSON structural errors.

Impact:
- Establishes the backend as the authoritative Source of Truth.
- Enables contextual data association for the upcoming Pass/Fail ML model.
- Provides trainers with granular control over the learning experience 
  directly through RESTful endpoints.
