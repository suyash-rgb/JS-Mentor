refactor: environment-aware student auto-creation

Refactored the "lazy upsert" workaround used for missing Clerk webhook synchronization during local development.

Instead of relying on a hardcoded TODO block that requires manual deletion before deployment, the system now dynamically checks the `FRONTEND_URL` environment variable. The automatic creation of placeholder student profiles will now ONLY execute if the frontend is running on `http://localhost`. 

This ensures that the development workaround safely disables itself in production environments where the actual webhook integration is expected to be functional.
