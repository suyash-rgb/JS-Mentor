This commit introduces significant UI improvements across the Learning Paths and the AI assistant page, alongside critical fixes for markdown rendering bugs, real-time mentorship chat synchronization, and file upload capabilities.

### 4. Real-Time Mentorship Chat Synchronization Fixes
- **Clerk Async Race Condition**: Fixed `useGlobalNotifications.js` where the `isSignedIn` check was executed synchronously on render before Clerk finished loading. Added a 500ms polling mechanism to wait for authentication, ensuring the Socket.IO connection is successfully established.
- **Chatbot Event Listeners**: Refactored `open-mentorship-chat` in `Chatbot.js` to remove the `isOpen` dependency, preventing race conditions where the listener was torn down precisely when an event fired.
- **AppRouter Integrations**: Wired up `force-open-chatbot` in `AppRouter.js` and added a pulsing red dot badge to the chatbot toggle button as a fallback notification if the chatbot doesn't pop open automatically.
- **Fallback Toasts**: Added a React Hot Toast notification in the student's chatbot when the trainer sends a message, ensuring visibility.

### 5. Dashboard Auto-Refresh Mechanisms
- **Trainer Dashboard (`StudentSupport.js`)**: Implemented a 30-second polling interval to auto-refresh the active doubt queue and sessions, eliminating the need for manual page reloads to see new doubts.
- **Student Dashboard (`Dashboard.js`)**: Implemented a 30-second polling interval and an immediate reload trigger tied to the `open-mentorship-chat` socket event, ensuring real-time session visibility.

### 6. Image Upload Fix
- **Authentication Gap**: Fixed an issue where image upload in `ChatBox.js` failed because `userToken` could be `null` when a student opens chat via the chatbot (Clerk's `getToken()` is async). Updated the upload handler to lazily resolve the token from `Clerk.session.getToken()` or `localStorage` as a fallback, ensuring the `Authorization: Bearer` header is never null. Replaced native `alert()` with `react-hot-toast` for a better user experience.
