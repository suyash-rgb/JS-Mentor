fix: configure native browser print for clean notes export

This commit optimizes the browser's native print functionality to provide a clean, zero-dependency export experience for curriculum notes, generating vector PDFs that preserve text selection, images, and Mermaid SVG diagrams without UI leakage.

[CSS Print Media Configuration]
- Added extensive `@media print` rules to `src/index.css`.
- Configured `-webkit-print-color-adjust: exact` to force correct color rendering for syntax highlighting and complex Mermaid SVG charts.
- Set standardized page rules (`@page { size: A4 portrait; margin: 2cm 1.5cm; }`) for consistent PDF layouts.
- Implemented defensive page-break logic (`page-break-after: avoid`, `page-break-inside: avoid`) to prevent headers from being orphaned and block elements (images, code blocks, tables, SVGs) from awkwardly splitting across pages.
- Enforced global hiding of interactive UI components (`.print:hidden`, `#chatbot-root`, etc.) to prevent them from leaking onto the printed canvas.

[Chatbot Leakage Fix]
- Updated `src/Router/AppRouter.js`: Appended the `print:hidden` utility class to the floating chatbot toggle button's container to ensure it disappears during export.
- Updated `src/components/chatbot/Chatbot.js`: Appended the `print:hidden` utility class directly to the chatbot popup window wrapper (`.chatbot-container`) so that an open chatbot dialog will not overlap the generated PDF.
