fix: resolve base64 image rendering and saving issues

This commit fixes multiple issues preventing Base64 compressed images from saving to the database and rendering in the markdown preview.

- React Markdown Sanitizer Bypass: 
  Updated `<ReactMarkdown>` components in both `NotesEditorPage.js` and `NotesViewerPage.js` to include a custom `urlTransform={(url) => url}` prop. This overrides the default strict sanitization behavior that was aggressively stripping out `data:image/jpeg;base64` URIs and causing empty `src=""` attributes (broken image icons).
- Fixed Invalid URL Injection Bug: 
  Corrected a bug inside `handleImageUpload` in `NotesEditorPage.js`. The generic markdown formatting function was accidentally injecting the placeholder word "text" directly into the image markup when no text was selected (creating invalid URLs like `![Image](textdata:image...)`). Replaced it with explicit string injection logic to correctly format `![Image](data:image...)` at the cursor's exact position.
